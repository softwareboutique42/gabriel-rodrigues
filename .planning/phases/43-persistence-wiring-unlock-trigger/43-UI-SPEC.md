> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Phase Context

Phase 43 is a persistence wiring phase. No new visual surfaces, pages, or components are introduced.
All UI rendering already exists in `src/pages/{en,pt}/casinocraftz/index.astro` and is driven by
`data-casinocraftz-*` dataset attributes emitted by the tutorial engine. This contract covers only
the interaction states that change as a result of the persistence fix — specifically lesson lock/unlock
rendering and the curriculum progress counter.

**No new markup, no new CSS classes, no new components are permitted.** All visual outcomes in
this phase are achieved exclusively by writing correct values to localStorage and having the engine
emit correct dataset state to the existing DOM.

Source: CONTEXT.md (deferred: no FX code, no Astro edits), RESEARCH.md (anti-patterns), REQUIREMENTS.md (EDU-70, EDU-71, EDU-73)

## Interaction States

These are the visual state changes that Phase 43 wiring must produce. All states are achieved
by the engine emitting correct `data-casinocraftz-*` dataset values — no CSS changes required.

### Lesson 3 Unlock State (EDU-70)

**Before fix (broken):** Returning user who completed Lesson 2 reloads page. Lesson 3 card renders
in locked state. `data-casinocraftz-unlocked-lessons` does not include `sensory-conditioning`.

**After fix (required):** Same user reloads page. `loadCompletedLessons()` reads
`ccz-near-miss-completed` from localStorage. Engine emits
`data-casinocraftz-unlocked-lessons="house-edge,near-miss,sensory-conditioning"`. Lesson 3 card
renders in unlocked state using the existing unlock CSS path.

**Visual outcome:** Lesson 3 card switches from locked style to unlocked style. No new CSS is
written — the existing render layer already handles both states.

### Curriculum Counter State (EDU-73)

**Before fix:** Counter reads `2/3` max for Lesson 2 completers (Lesson 3 was never reachable).

**After fix (required):** User who completes Lesson 3 and reloads sees `data-casinocraftz-completed-lessons`
emit `"house-edge,near-miss,sensory-conditioning"`. Existing render layer shows `3/3`.

**Visual outcome:** Counter text updates from `2/3` to `3/3`. No new CSS.

### Skip Path Persistence (EDU-70, EDU-73)

**Required:** When user skips Lesson 3, the skip handler writes `ccz-near-miss-completed` before
`completeCurrentLesson()`. On reload, Lesson 3 appears completed, not locked.

**Visual outcome:** Identical to the normal completion path — Lesson 3 card renders completed.

### Lesson 3 Dialogue Steps (EDU-71)

All four sensory-conditioning dialogue steps are pre-authored and already render correctly via
the existing dialogue system. No visual change is needed — this requirement is satisfied by
confirming the existing render path reaches the steps, not by adding new UI.

These constraints are derived from upstream decisions and apply to all executor work in Phase 43:

1. **No Astro page edits** — `src/pages/{en,pt}/casinocraftz/index.astro` must not be modified.

   references of any kind. FX belongs to Phases 45-46 (v2.3).
4. **No JSON in localStorage** — Use `'1'` flag strings only, read via `=== null` presence check.
5. **No runtime contract tests** — All new test assertions use `readWorkspaceFile()` + `assert.match()`.
