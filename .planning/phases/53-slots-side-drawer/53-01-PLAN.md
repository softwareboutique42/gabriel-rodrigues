<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| keyboard input -> drawer controller | Untrusted key events can trigger open/close/focus logic |
| sessionStorage -> UI state | Browser-stored value controls initial drawer open state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-53-01 | Tampering | `sessionStorage` drawer state in `src/scripts/slots/main.ts` | mitigate | Validate stored value strictly (`'true'`/`'false'`) and fallback closed on invalid value |
| T-53-02 | Denial of Service | focus trap handler in `src/scripts/slots/main.ts` | mitigate | Scope trap only when drawer open and menu exists; guard missing focusables and exit cleanly |
| T-53-03 | Spoofing | drawer trigger state (`aria-expanded` + icon) | mitigate | Single `setOpen()` source of truth updates dataset, aria, and icon atomically |
| T-53-04 | Information Disclosure | drawer content | accept | Drawer displays static educational copy only, no secrets or user PII |
</threat_model>
